# 音效设计规范 v1

> 维护者：SD | 最后更新：2026-05-10
>
> 本文档为游戏音频体验的唯一权威参考。ND 标记音效、FD 调用接口、QA 验收音效均以此为准。

---

## 1. 设计哲学

| 原则 | 说明 |
|------|------|
| **极简** | 每章 ≤ 3 种声音事件。少即是多 |
| **低频** | 所有氛围音 ≤ 80Hz。低频让玩家感到不安而不确定原因 |
| **留白** | 沉默本身是音效。章节过渡、选项出现、结局判定——主动静默 > 持续铺底 |
| **叙事优先** | 音效服务于文本焦虑，不抢注意力。永远不会在逐字打印期间引入新音效 |
| **零依赖** | 全部代码合成（wave + struct + math），不依赖外部 .wav/.mp3 素材文件 |

## 2. 音效注册表

### hum_low · 低频嗡声

| 属性 | 值 |
|------|-----|
| 频率 | 40Hz 正弦波 |
| 时长 | 循环 |
| 音量 | 0.3（基底），可叠加 |
| 触发 | day ≥ 3 夜间节点自动触发 |
| 说明 | 模仿塔身混凝土的共振频率。玩家不"听到"它，但移除时会感到突然的不安 |

### heartbeat · 心跳

| 属性 | 值 |
|------|-----|
| 频率 | 60Hz 双脉冲（"咚咚"） |
| 时长 | 循环，2Hz 节奏 |
| 音量 | 0.5 |
| 触发 | [shake] 红色抖动文字同步触发；黑暗收缩期间触发 |
| 说明 | 不标注化心跳——略慢于正常心率，让玩家感到自己的心跳"被跟踪" |

### metal_creak · 金属收缩

| 属性 | 值 |
|------|-----|
| 频率 | 800Hz 高频方波 |
| 时长 | 0.3s 单次 |
| 音量 | 0.4 |
| 触发 | ND 标记 `[sound metal_creak]` |
| 说明 | 模仿灯塔金属结构热胀冷缩。只在特定剧情节点标记使用，每章 ≤1 次 |

### water_drone · 水下嗡鸣

| 属性 | 值 |
|------|-----|
| 频率 | 30Hz 滤波白噪声 |
| 时长 | 循环 |
| 音量 | 0.2 |
| 触发 | 凌晨/夜晚水下场景自动触发（chapter ≥ 2, day ≥ 3, 时间标记为夜间） |
| 说明 | 低音量环境底噪。不是"海浪声"——是"水压"的感觉 |

### silence_fade · 静默淡出

| 属性 | 值 |
|------|-----|
| 频率 | 0 |
| 时长 | 0.5s 线性淡出 |
| 音量 | 从当前值 → 0 |
| 触发 | 节点切换、章节过渡、面板打开时自动触发 |
| 说明 | 不是独立音效，是停止当前音效的过渡。避免突兀切断 |

## 3. 静默规则表

| # | 条件 | 行为 | 实现位置 |
|---|------|------|----------|
| 1 | 节点切换 / goto_node 被调用 | 当前音效 → silence_fade 0.5s → 判定新节点是否需要新音效 | story_engine.py |
| 2 | 面板打开（Tab 键） | 当前音效 → silence_fade 到 0；面板关闭后恢复（若该节点仍应播放音效） | main_window.py / main.py |
| 3 | 选项出现（_on_text_done） | 音量 → 当前值的 20%；选择后（cmd_choice）恢复 | main.py |
| 4 | 结局判定 / 死亡 / 未完待工 | 全部音效 → stop_all → 静默 1.5s → 文本浮现 | main.py |

## 4. JSON 标记规范

### 4.1 事件音效标记

ND 在 JSON `text` 字段中使用 `[sound name]` 标记事件音效的触发点和结束点：

```
"text": "你听到了什么。\n\n[sound metal_creak]\n塔身在响。\n\n它越来越近了。[/sound]"
```

| 语法 | 说明 |
|------|------|
| `[sound name]` | 触发指定音效 |
| `[/sound]` | 停止当前事件音效 |

### 4.2 生命周期规则

**`[sound]` 的生命周期与所在 `\n\n` 文本分块绑定**：
- 音效在块开始时触发
- 音效在块结束时自动停止（`[/sound]` 可省略，引擎自动在块尾 stop）
- 若 `[sound]` 和 `[/sound]` 跨块，以 `[/sound]` 显式标记为准

### 4.3 ND 标记原则

| 类型 | 标记方式 | 示例 |
|------|----------|------|
| **事件音效** | ND 手动 `[sound name]` | `[sound metal_creak]`、`[sound heartbeat]` |
| **环境音效** | 引擎自动化，ND 不标记 | hum_low / water_drone 由 chapter + day 自动判定 |

## 5. 自动化触发逻辑

以下音效由引擎根据节点属性自动判定，ND 无需在 JSON 中手动标记：

```python
# 伪代码（在 story_engine.goto_node 或 main._display_node 中执行）

def _auto_sound(node):
    if node.day >= 3 and node.chapter >= 2:
        # 夜间节点 → 低频嗡声
        if "夜晚" in node.text or "深夜" in node.text or node.day >= 3:
            sound_mgr.play("hum_low")
    if node.day >= 3 and ("水下" in node.text or "海" in node.text):
        sound_mgr.play("water_drone")
```

## 6. 存档格式

`save_manager.py` 中 save 字典增加 `sound_state` 字段：

```json
{
  "sound_state": {
    "active": "hum_low",
    "volume": 0.8,
    "muted": false
  }
}
```

读档时调用 `SoundManager.from_dict(data["sound_state"])` 恢复音效状态。

## 7. 技术约束

| 约束 | 说明 |
|------|------|
| **异步播放** | 所有 `play()` 必须在独立线程中执行，不阻塞 Tkinter 主线程（逐字动画不能卡） |
| **预生成缓存** | `SoundManager.__init__` 时一次性合成全部 5 个 `.wav` 到 `data/sounds/`，后续 `play()` 只读缓存 |
| **无外部文件** | exe 打包后不依赖任何外部 .wav/.mp3——全部缓存文件在运行时生成到临时目录 |
| **线程安全** | `play`/`stop`/`mute` 操作必须持有 `threading.Lock` |
