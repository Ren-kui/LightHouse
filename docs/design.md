# 第一卷 · 交互逻辑

> 认领角色：**UI**（主责）、FD（配合实现）
>
> [UI] 我负责玩家看到什么、摸到什么、怎么摸。标题界面→逐字文本→箭头选项→滑入面板，四块全部对齐Undertale设计哲学。黑底白字是底色，橙红是唯一强调色。逐字打印让玩家与文本同步呼吸。空格继续是给人类的提示。Tab滑入面板代替弹窗。

---

## 1.1 进入游戏流程

双击启动 -> 引语淡入 -> "灯  塔"标题浮现 -> 1.5秒 -> 菜单：新游戏/继续游戏/退出 -> 第1章逐字打印



## 1.2 核心交互界面

### 标题界面
- 引语(灰色小字) + "灯 塔"(红色标题) + 居中菜单
- 悬停亮橙(#cc6600)，点击触发流程

### 文本阅读界面
- 逐块逐字打印，纯黑底+微软雅黑粗体白色字
- 空格/回车/点击：打字中跳过当前块 / 块结束翻页 / 全部结束显示选项
- 提示："▼ 空格/回车继续"(灰色)

### 选项界面
- ▶ 选项文字，居中排列
- 悬停箭头白+文字橙，点击执行选择

### 状态面板(Tab呼出)
- 主窗口变暗 → 右侧滑入32%宽度，笔记风格隐晦状态+物品悬停

### 存档/读档
- 纯黑浮窗，3手动+1自动，▶ 箭头选项

## 1.4 UI动画

### 逐字打印
- 30ms/字，按\n\n分块，块间按键翻页

### 红色抖动文字
- #cc0000 + 水平+-3px偏移 12.5次/秒，持续5秒，每章<=3次

### 多次点击按钮
- 连点3-5次，文字逐次变化，颜色渐红

### 黑暗收缩
- V1矩形边框50秒内收，失败->全黑->1.5秒留白->死亡文本浮现

# 第二卷 · 功能与玩法设计

> 认领角色：**GD**（主责）
>
> [GD] 玩法核心是选择改变命运。3显性变量影响选项出现消失，2隐藏变量在关键节点引爆。小游戏升级为叙事工具(黑暗收缩失败即死)。5结局加特殊死亡，变量组合唯一判定。

### 判定优先级

### 结局触发条件（GD + ND 对齐 · 2026-05-11）

| 优先级 | ID | 结局 | 判定条件 | 可达路径 |
|--------|----|------|---------|---------|
| 1 | **G** | 荒诞 · 被带走 | flags: found_diary_page + found_wangchao_drawing + found_dad_logbook 全 True | curiosity≥7（水彩画）+ survival_will≥8（日志）+ loyalty≥4（撕页） |

| 4 | **A** | 疯狂 · 独自对话 | curiosity≥7 + sanity≤3 + (trust≥5 或 loyalty≥3) | 高好奇但留了一条人际关系 |
| 5 | **E** | 提前逃离 · 永不知答案 | survival_will≥8 + sanity≤4 + trust≤4（ch05 特殊节点） | 高生存 + 低信任 |

## 4.5 音效模块 (sound_manager.py)

> 认领角色：**SD**（主责）、FD（挂载接口）
>
> [SD] 极简主义：每章 ≤3 种声音事件。低频 + 留白 > 频繁打断。音效服务于叙事焦虑，不抢文本注意力。全部纯代码合成（wave + struct），零外部依赖/零音频素材文件。

### API 接口

| 方法 | 说明 |
|------|------|
| `play(name)` | 播放命名音效（如 "hum_low"、"heartbeat"） |
| `stop(name)` | 停止指定音效 |
| `stop_all()` | 静默全部音效 |
| `set_volume(0.0~1.0)` | 全局音量控制，带边界保护 |
| `mute()` / `unmute()` | 静音开关 |

### 音效注册表

| 名称 | 频率/波形 | 时长 | 循环 | 触发场景 |
|------|-----------|------|------|----------|
| `hum_low` | 40Hz 正弦波 | 循环 | 是 | day≥3 夜间节点自动触发，底噪氛围 |
| `heartbeat` | 双脉冲 2Hz | 循环 | 是 | [shake] 红色抖动同步，黑暗收缩期间 |
| `metal_creak` | 高频方波 | 0.3s | 否 | 灯塔金属收缩，特定剧情节点标记 |
| `water_drone` | 滤波白噪声 | 循环 | 是 | 凌晨/夜晚水下场景，低音量环境底噪 |
| `silence_fade` | 当前音效淡出 | 0.5s | 否 | 节点切换、章节过渡、面板打开时触发 |

### 静默规则

| # | 条件 | 行为 |
|---|------|------|
| 1 | 章节过渡 / 节点切换 | `silence_fade` 淡出旧音效 0.5s |
| 2 | 面板打开（Tab） | 当前音效淡出到 0，面板关闭后恢复 |
| 3 | 选项出现 | 音量降至 20%，选择后恢复 |
| 4 | 结局判定 / 死亡 | 全部音效立即停止，静默 1.5s → 文本浮现 |

### 自动化触发

环境音效（`hum_low`、`water_drone`）由引擎根据当前 chapter/day 自动判定触发，ND 无需在 JSON 中手动标记。事件音效（`heartbeat`、`metal_creak`）由 ND 在 JSON text 中使用 `[sound name]` 标记，生命周期与所在 `\n\n` 文本分块绑定——块内触发、块外停止。

### 存档格式

```json
"sound_state": {
  "active": "hum_low",
  "volume": 0.8,
  "muted": false
}
```
