# 文本标记规范 v1

> 维护者：FD | 最后更新：2026-05-09
>
> 给 ND 的 JSON 文本编写规范。所有标记在 `text` 字段中使用。

---

## 1. 自然分块（已有）

文本中的 `\n\n`（两个连续换行，中间可含空格）会被引擎自动识别为块边界。逐字打印时，每个块独立打印，块间等待玩家按键翻页。

```json
"text": "第一段内容。\n\n第二段内容。\n\n第三段内容。"
```

> **规则**：一个块 = 一个自然段落。不要用 `\n\n` 把一句话切成两半。

---

## 2. `[pause N]` 停顿标记

在文本中插入 `[pause N]`，表示当前块打印完毕后额外停顿 N 秒再显示翻页提示。

```json
"text": "你推开门。\n\n[pause 1.5]\n\n海风很冷。"
```

- N 为秒数，支持小数（如 0.5、1.5）
- 标记放在块开头或块结尾均可
- 引擎会在当前块打印完后等待 N 秒再显示 "▼ 空格/回车继续"

---

## 3. `[shake]` 红色抖动标记

将一段文字标记为红色抖动效果。用 `[shake]` 开始、`[/shake]` 结束。

```json
"text": "海浪声里……有声音。\n\n[shake]不是风声。是有人在说话。[/shake]\n\n你握紧了栏杆。"
```

- 标记放在一个块内部
- 被包裹的文字显示为红色（#cc0000），并水平随机抖动 5 秒
- 每章使用 ≤ 3 次

---

## 4. 多击选项 `multi_click`

在 `choices` 的选项中添加 `multi_click` 字段，玩家需连点 N 次才能生效。

```json
"choices": [{
  "label": "打开那扇门",
  "next_node": "ch04_door_open",
  "effects": {"curiosity": 2, "sanity": -1},
  "conditions": null,
  "multi_click": {
    "count": 4,
    "texts": ["我再想想……", "手在发抖，做不到……", "还是抓住了门把手……", "推开了"]
  }
}]
```

- `count`：需要点击的次数
- `texts`：每次点击后按钮上显示的文字列表（长度 = count）
- 最后一次点击后执行选项效果 + 跳转
- 按钮颜色逐步变红

---

## 5. 逃避对白

小游戏前的"自己做 / 让队友代劳"选择，用两个独立的 `choices` 条目表示。

```json
"choices": [
  {
    "label": "自己完成配电连线",
    "next_node": "ch02_minigame_wire",
    "effects": {},
    "conditions": null
  },
  {
    "label": "让张海生来做",
    "next_node": "ch02_work_escape_01",
    "effects": {"trust": -1},
    "conditions": null
  }
]
```

- 逃避选项的 `next_node` 指向一个纯剧情节点（ND 写逃避后的对话）
- `effects` 记录关系变化
- 小游戏实现本身由 FD 在 `next_node` 中接入（里程碑 5），ND 只需写好前后的过渡文本

## 7. `[key]...[/key]` 关键句标记（v2）

段落中的关键语句或关键词用 `[key]...[/key]` 包裹。引擎渲染时以淡黄色（#ccaa44）显示，不加粗不倾斜，仅改变文字底色感，帮助跳读型玩家快速抓住剧情重点。

```json
"text": "你在木架最底层看到了[key]一个八音盒[/key]。没有发条的。"
```

**规则**：
- 每个节点 1-3 处，不超过 3 处
- 标记短语长度：2-20 字
- 不标记 UI 文本（label 字段）、不在 `[shake]` 内嵌套
- ND 自检：标注后自己跳读一遍，确认只看黄色文字能理解本章剧情走向

---


## 5. JSON 文本引号规范

**规则**：`text` / `label` / `desc` 等文本字段内禁止使用未转义的 ASCII 双引号 `"`。必须使用中文引号 `""`（U+201C / U+201D）。

```json
// ❌ 错误 — 内嵌 ASCII " 破坏 JSON 结构
"text": "他说"没有任务"。"

// ✅ 正确 — 使用中文引号
"text": "他说\u201c没有任务\u201d。"
```

**依据**：B025（diary.json）和 B029（chapter_02.json）均为 ASCII `"` 混入 JSON 文本值导致解析失败。JSON 标准要求文本值内的 `"` 必须转义为 `\"`，但中文文本用 `\u201c\u201d` 更自然。

> ND 自检项：写完章节 JSON 后运行 `python tests/validate_data.py`，确认无 JSON 解析错误。


## 6. 标记歧义消除

**规则**：同一标记符号不得在不同视觉状态间复用。每个需要不同渲染效果的状态必须有独立的标记符号。

| 标记 | 含义 | 颜色 | 来源 |
|------|------|------|------|
| `‡...‡` | 变量低于阈值（不祥/恶化） | `#cc0000` 红 | B031 修复 |
| `†...†` | 变量高于阈值（稳定/正常） | `#008800` 绿 | B031 修复 |

**依据**：B031 — diary.json 中 low/high 阈值共用 `‡` 标记，引擎只能渲染一种颜色，改变颜色后高低阈值全部变红。标记的视觉含义必须与数据语义一一对应。

> ND 自检项：新增标记前检查 `text_markup_spec.md` 避免与已有标记语义冲突。
