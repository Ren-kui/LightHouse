> 未填写本清单的交付 = QA 拒收（依据 `work_standards.md` W003 ND 自检声明）。

---

## 使用方式

1. **新章自检**：`python src/main.py --gm` → Ctrl+Shift+G → GM 加载本章 → 逐分支测试。
2. **回顾重检（如第 1-3 章修复断桥）**：从 `启动游戏.bat` 开始，引语 → 标题 → 新游戏，完整走完所有已写章节，捕捉章与章之间的过渡断裂。GM 跳章会漏掉跨章承接。
3. 空格翻页从头读到尾，**亲手选每一个选项**（含逃避/沉默/反问等次选路径）。
4. 逐项填入结果（PASS / FAIL）。
5. FAIL = 修完该节点后重检。
6. 将本表 + 变更通知（节点数、标记类型与数量）一并交付 QA。
7. **同源异构专项扫描**：运行 `python tests/validate_data.py`，查看 `[汇聚节点报告]` 列出的每个节点。逐项对照 `docs/bridges.md` 汇聚节点配置表——每个汇聚节点的每条来路都必须有独立桥段或归于 `*` fallback。缺桥 = 未完成。修桥时遵循 `bridges.md` §2 写作原则（事件/时间/空间/情感连续性，50-300 字）。

## 自检表

| # | 检查项 | 说明 | 结果 |
|---|--------|------|------|
| 1 | 全部分支覆盖 | 从头走完主流 + 全部次选（逃避/沉默/反问/拒绝/独自行动等），无一跳过 | PASS |
| 2A | 时间连续性 | 选择→下一块之间的时间跨度不要太跳跃；场景切换时是否有一句时间过渡 | PASS |
| 2B | 动作连续性 | 选择描述的动作（阅读/追问/探索）是否在下一块文本中得到了起码一句承接 | PASS |
| 2C | 人物状态一致性 | 多路径汇聚节点的文本没有假设玩家做了某个特定动作——GM 逐路跳节确认每条来路的玩家都能在文本中找到自己 | PASS |
| 3 | 选项目标可用 | 不显示不应出现的选项——条件未满足的选项被正确过滤 | PASS |
| 4 | [pause] 时长匹配情绪 | 句末停顿——1.5s 是欲言又止，2.0s 是重大沉默；没有无故的短/长 pause | PASS |
| 5 | [shake] 文本 ≥ 8 字 | UI 已验证短文本抖动视觉弱（3 字以下不建议标 shake） | PASS |
| 6 | 节奏"两紧一松" | 本章不能持续高压——紧张节点和日常/过渡节点交错；高压段不连续超过 3 个节点 | PASS |
| 7 | 节点间有承接 | 每个节点的文本至少有一句呼应上一段的尾句或选择。汇聚节点必须有 `text_bridges`：对照 `docs/bridges.md`，每条来路配独立桥或归 `*`。 | PASS |
| 8 | 本章情绪的峰值和终值 | 最紧张的一段在哪？最后一段留给读者的感受是什么？有没有草草收尾？ | PASS |
| 9 | 同源异构路径状态一致 | GM 逐路跳节每条来路 → 读目标节点前 3 句 → 问："我做了这件事吗？" 答案不是"是"= 断桥。断桥处参照 `docs/bridges.md` §2 修。 | PASS |
| 10 | 句式多样性 | 一个节点内"像……""不是……是……""比……更……"类比喻句式各不超过 1 处；白描不是不用修辞，是用一次就不再用第二次 | — |
| 11 | 新增 effects 终点可达性 | 每在已对齐 effects 表之外新增一个 effects 影响终值时，GM 跳节模拟该路径终值，确认对应结局阈值仍在可达范围内 | — |
