## D001 - 技术选型

| 项 | 决定 |
|---|------|
| **日期** | 2026-05-08 |
| **决定** | 采用 Python + Tkinter 开发桌面客户端 |
| **理由** | Python 环境已具备；Tkinter 原生桌面、轻量、可打包为 exe；符合"非网页游戏"需求；开发复杂度与项目体量匹配 |
| **备选方案** | Node.js + Electron（体积过大，被否决） |

---

## D002 - 分支模式

| 项 | 决定 |
|---|------|
| **日期** | 2026-05-08 |
| **决定** | 采用模式 B：带状态变量的分支系统 |
| **理由** | 用少量节点组合出丰富变化，支持直接分流、阈值判定、组合判定三种节点类型；维护成本低于纯树形分支 |
| **节点类型** | ① 直接分流（选A去A页，选B去B页） ② 阈值判定（变量 >= N 时解锁某选项） ③ 组合判定（多变量同时满足条件时触发） |

## D015 - 小游戏叙事化：黑暗收缩死亡场景

| 项 | 决定 |
|---|------|
| **日期** | 2026-05-08 |
| **决定** | 小游戏不仅用于调剂，更承载叙事和决定游戏走向的功能。首个应用场景：停电 + 阴影吞噬光源 + 玩家跑向 4 楼重启供电。 |
| **具体规格** | 界面边缘黑暗矩形向内收缩（50秒倒计时），小游戏居中不受覆盖；失败 → 角色死亡，触发特殊死亡结局 |
| **死亡呈现方式** | 同一界面分两阶段：①收缩动画 + 小游戏；②黑幕全覆盖 → 留白 1-2 秒 → 血红/白色文字渐显，死前独白与画面描述合为一段（3-5 句） |
| **死亡结局归类** | 特殊死亡结局单列为"灯塔未解死亡报告"，与 5 条主线结局平行 |
| **技术实现** | 封装 `DarknessOverlay` 类；V1 矩形收缩 → 效果不满意可升级 V2/V3/V4 |
| **文字浮现方式** | 逐字打印，速度略慢于正常阅读 |

## D016 - 黑暗收缩效果技术路线

| 项 | 决定 |
|---|------|
| **日期** | 2026-05-08 |
| **V1（基线）** | 单一矩形边框向内收缩，规整流畅 |
| **V2** | 每边独立收缩速度，产生不对称挤压感 |
| **V3** | 多块小矩形随机分布在边缘，模拟"啃咬吞噬" |
| **V4** | 预渲染 PNG 遮罩动画帧逐帧替换，最精致 |
| **架构原则** | `DarknessOverlay` 类独立封装，内部实现可随时替换，不影响游戏逻辑 |

## D029 - SD 音效设计师加入 + 音效基础架构

| 项 | 决定 |
|---|------|
| **日期** | 2026-05-10 |
| **决定** | 新增 SD（音效设计）角色。团队由 6 人扩充为 7 人编制（PM/GD/ND/FD/UI/QA/SD）。SD 负责全部音频体验设计，技术路线纯代码合成（wave + struct），零外部依赖/零素材文件。SD 不修改游戏逻辑代码，通过 sound_manager.py API 提供接口 |
| **职责** | 氛围音 + 事件音 + 静默设计。每章 ≤3 种声音事件；低频 + 留白 > 频繁打断；音效服务于叙事焦虑 |
| **M4 交付** | `src/sound_manager.py` 空壳 API（play/stop/stop_all/set_volume/mute/unmute）；`docs/sound_spec.md` 规范文档（含静默规则表）；`data/sounds/` 目录占位；QA 基础验收（SD-01~04） |
| **M5 交付** | 5 核心音效实现（hum_low/heartbeat/metal_creak/water_drone/silence_fade）；`[sound]` 标记引擎解析；各章节音效触发点标注 |
| **M6 交付** | 音效时序调优 + 音量控制 + exe 打包验证 |
| **技术决策** | ① 预生成策略——所有音效在 SoundManager.__init__ 时一次性合成到 data/sounds/ 缓存，避免实时合成 I/O 延迟 ② 异步播放——不阻塞 Tkinter 主线程，保障逐字动画流畅 ③ 存档预留——save 字典增加 sound_state 字段（M4 空对象占位）④ 静默规则——章节过渡强制静默 0.5s、面板打开淡出到 0、选项出现音量降至 20% ⑤ 环境音效自动化——夜间节点根据 chapter/day 自动触发，ND 只标记事件音效 ⑥ `[sound]` 生命周期与 `\n\n` 文本分块绑定——块内触发、块外停止 |
