## 1. 桥是什么

桥是放在汇聚节点 `text` 主体之前的、**按来路区分**的叙事垫片（存放在 JSON 的 `text_bridges` 字段中，由 `story_engine.py get_current_text()` 自动拼接）。

**桥解决的问题**：当多条路径（不同的选择）汇聚到同一个节点时，如果没有桥，无论玩家选了哪个选项，下一段正文都是相同的——这意味着玩家的选择没有获得叙事反馈，造成以下四种断裂：

| 连续性 | 断裂表现 | 桥如何修复 |
|--------|---------|-----------|
| **事件连续性** | 选了"读手册"但下一段直接到厨房吃饭 | 写读手册→合上书→起身去厨房 |
| **时间连续性** | 选择发生在下午但下一段是深夜 | 写天色变暗的过程 |
| **空间连续性** | 选择在配电室但下一段在厨房 | 写从配电室走到厨房 |
| **情感连续性** | 选择时好奇紧张但下一段平淡如水 | 延续并过渡情绪 |

**简单说**：桥 = 兑现玩家点击选项时的期待。

**为什么需要桥**：本游戏的核心循环是固定的主线 + 变量分叉（design.md §2.1）。选项改变变量值，但所有路径最终汇聚回同一条主线节点。没有桥 = 选择后话不对版，所选非所得。有桥 = 不同路径有不同过渡，但汇聚点自然。

---

## 桥的两类作用

| | 类型一 · 分叉叙事桥 | 类型二 · 体验过渡桥 |
|--|-------------------|-------------------|
| **用** | 选择产生不同的叙事结果（比如问了得到答复 vs 没问没得到答复），后续正文中存在与某条路径冲突的内容 | 选择不产生不同的叙事结果（两种选择最终都是"没有发生什么"），正文对两条路径都成立 |
| **写** | 将正文中会产生冲突的分叉内容**移入桥**。回复、对话、揭示——属于哪条路径就放在哪条路径的桥里。正文只保留两条路径都成立的部分 | 50-100 字，只描述"你怎么度过这段时间"的体验差异。不写对话、不写揭示、不写情节转折 |
| **例** | 问了螺丝 → 桥写"他说'热胀冷缩'"；没问 → 桥写"你没有开口" | 下床检查 → 桥写"在走廊走了几步，回床睡了"；强迫睡觉 → 桥写"翻来覆去，勉强睡着" |
| **规模** | 可达 200-400 字 — 需要承载实质叙事内容 | 50-100 字 — 只需简述体验差异 |
| **正文从何起** | 两条路径的共识时间/空间点 | 同左 |

**判断方法**：读完目标节点的正文第一段 → 问"如果我从这条路径来的——这段话成立吗？" 不成立的部分就是**需要放进桥里的分叉内容**。

## 2. 写作原则

| # | 原则 | 说明 |
|---|------|------|
| 1 | **事件连续性** | 桥必须兑现选择动作。选了"翻抽屉"就写翻抽屉的内容，选了"追问"就写追问的场景。 |
| 2 | **时间连续性** | 从选择发生的时刻自然过渡到目标节点时刻。用光线变化、身体感受、环境音暗示时间流逝。 |
| 3 | **空间连续性** | 从选择发生的空间自然过渡到目标节点空间。写移动、写走廊、写脚步。 |
| 4 | **情感连续性** | 选择带来的情绪（好奇/紧张/疏远/信任）必须在桥中得到延续或过渡，不能丢失。 |
| 5 | **篇幅** | 50-400 字。一句以上，一段以内。不是连接词，是**叙事垫片**。 |
| 6 | **白描** | 保持 ND P1 原则：白描 > 渲染，暗示 > 直说。详见 `docs/agents.md` ND 角色行。 |
| 7 | **句式多样性** | 服从 ND P6：桥段内"像/不是…是…/比…更…"各 ≤1 处。 |
| 8 | **不剧透** | 桥段中不得包含任何尚未发生的情节信息或变量变化暗示。 |
| 9 | **不预演正文** | 桥段只负责过渡（把玩家从选择时刻带到新场景门口）。新场景的第一个事件、第一句话由正文全权负责。桥段抢正文的首句 = 玩家听到同一件事两次。桥段应止于时间/空间/情绪承接，正文从场景动作开始。 |
| 10 | **桥的时间线必须停在正文之前** | 桥只能写到"然后你去了厨房"——不能写"你在厨房看到了张海生，他说了什么"。对话、NPC 反应、新信息揭示、情节转折——属于正文。类型一桥除外：分叉内容（如"他答了A / 他没答"）必须在进入共识正文前交代完毕，但仍不可预演正文的后续场景。 |

### 黄金规则

> **桥只回答"怎么到的"，不回答"路上发生了什么"。** 如果需要"路上发生了什么"——那就是类型一分叉叙事，把它写进桥，但**从正文中删掉不再属于共识路径的部分**。

### `*` fallback 写作原则

当玩家从某条未配置独立桥的路径到达汇聚节点时，`*` fallback 兜底。要求：不提任何具体选择内容，仅以时间/空间/状态过渡：

> "下午在塔里各自忙开了。不管你是追上去问过温度计的事、还是盯着他那根不听话的无名指、又或者一个人翻了几页维护手册——等你回过神来的时候，海面已经沉进了暮色里。"
